## 📧 NEW FEATURE: Renewal Reminder Emails

**What It Does**: Automatically sends reminder emails 7 days before subscription expires

### Database Changes

**Migration**: `add_renewal_reminder_fields`

**New Columns in `payments` table**:
- `purchase_date` - When purchase was made
- `subscription_end_date` - Auto-calculated (purchase + 365 days)
- `renewal_reminder_sent` - Boolean flag
- `renewal_reminder_sent_at` - When reminder was sent

### New Edge Functions

**1. send-renewal-reminder**
- Finds subscriptions ending in 7 days
- Sends personalized HTML emails
- Includes 10% discount code (RENEW10)
- Marks reminders as sent

**2. check-renewal-reminders-cron**
- Wrapper for scheduled execution
- Called by cron job daily
- Triggers send-renewal-reminder

### Email Features

- **Timing**: 7 days before expiry
- **Personalization**: Recipient name + theme
- **Incentive**: 10% discount (RENEW10)
- **Branding**: Professional HTML template
- **Currency**: Shows AUD pricing

### Setup Required

**1. Resend Account** (2 minutes):
- Sign up at https://resend.com
- Get API key
- Add to Supabase secrets as `RESEND_API_KEY`

**2. Cron Scheduler** (1 minute):
- Option A: GitHub Actions (recommended)
- Option B: External cron service (cron-job.org)
- Option C: Supabase pg_cron

**Documentation**:
- Quick Start: `RENEWAL_REMINDER_QUICKSTART.txt`
- Detailed Guide: `RENEWAL_REMINDER_SETUP.txt`

## 📊 MONITORING

### Check Renewal Reminders

```sql
-- Count reminders sent
SELECT COUNT(*) FROM payments WHERE renewal_reminder_sent = true;

-- Upcoming reminders
SELECT * FROM payments 
WHERE renewal_reminder_sent = false 
AND subscription_end_date BETWEEN NOW() AND NOW() + INTERVAL '7 days';
```

### Check Email Delivery
