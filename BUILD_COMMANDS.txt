
# Daily Whispers - Quick Build Commands Reference

## Prerequisites
```bash
# Install EAS CLI globally
npm install -g eas-cli

# Login to Expo
eas login

# Initialize EAS project (first time only)
eas init
```

## Development Builds

### iOS Development
```bash
eas build --platform ios --profile development
```

### Android Development
```bash
eas build --platform android --profile development
```

## Preview Builds (Internal Testing)

### iOS Preview
```bash
eas build --platform ios --profile preview
```

### Android Preview
```bash
eas build --platform android --profile preview
```

## Production Builds

### iOS Production (App Store)
```bash
# Build
eas build --platform ios --profile production

# Submit to App Store
eas submit --platform ios --profile production
```

### Android Production (Google Play - AAB)
```bash
# Build AAB (required for Play Store)
eas build --platform android --profile production-aab

# Submit to Google Play
eas submit --platform android --profile production
```

### Android Production (APK for direct distribution)
```bash
eas build --platform android --profile production
```

## Build Both Platforms Simultaneously
```bash
eas build --platform all --profile production
```

## Check Build Status
```bash
eas build:list
```

## View Build Details
```bash
eas build:view [BUILD_ID]
```

## Download Build
```bash
eas build:download [BUILD_ID]
```

## Update Credentials

### iOS Credentials
```bash
eas credentials
```

### Android Keystore
```bash
eas credentials -p android
```

## Testing

### Install on iOS Device (Development Build)
```bash

```

### Install on Android Device (Development Build)

## Build Profiles Explained

### development
- Creates a development client
- Includes debugging tools
- Larger file size
- For internal testing only

### preview
- Similar to production but for internal distribution
- Good for beta testing
- Smaller than development builds

### production
- Optimized for app stores
- Smallest file size
- No debugging tools
- Ready for public release

### production-aab
- Android App Bundle format
- Required for Google Play Store
- Optimized for different device configurations

## Important Notes

1. **First Build**: The first build will take longer as it sets up credentials
2. **iOS Builds**: Require Apple Developer account ($99/year)
3. **Android Builds**: First build generates keystore - SAVE IT SECURELY!
4. **Build Time**: Typically 10-20 minutes per platform
5. **Concurrent Builds**: Free tier allows 1 concurrent build
6. **Build History**: Builds are stored for 30 days

## Before Building

1. Update version in app.json
2. Update buildNumber (iOS) and versionCode (Android)
3. Ensure all dependencies are installed
4. Test locally first
5. Commit all changes to git

## After Building

1. Download and test the build
2. Submit to app stores
3. Monitor review status
4. Respond to any review feedback

## Common Issues

### "Build failed" - Check logs for specific error
```bash
eas build:view [BUILD_ID]
```

### "Credentials expired" - Update credentials
```bash
eas credentials
```

### "Bundle identifier already exists" - Change in app.json
```json
{
  "ios": {
    "bundleIdentifier": "com.yourcompany.uniquename"
  }
}
```

### "Keystore not found" - Generate new keystore
```bash
eas credentials -p android
```
