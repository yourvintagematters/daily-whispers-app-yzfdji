
# Daily Whispers - Production Deployment Guide

- [ ] Google Play Console Account ($25 one-time) - https://play.google.com/console

### 2. Stripe Configuration
- [x] Live Stripe publishable key configured in utils/paymentConfig.ts

### 3. App Assets Required
- [ ] High-resolution app icon (1024x1024 PNG)
- [ ] Splash screen image

- [ ] Google Play screenshots (various Android device sizes)

## Step-by-Step Deployment Process

### Phase 1: Configure Supabase for Production

1. **Add Stripe Secret Key to Supabase**
   - Go to: https://supabase.com/dashboard/project/rfaqhkbbyvddaxyolxrx/settings/secrets
   - Add secret: STRIPE_SECRET_KEY = sk_live_YOUR_SECRET_KEY
   - Restart Edge Functions after adding the secret

2. **Configure Stripe Webhook**
   - In Stripe Dashboard, go to Developers > Webhooks
   - Add endpoint: https://rfaqhkbbyvddaxyolxrx.supabase.co/functions/v1/stripe-webhook
   - Select events: payment_intent.succeeded, payment_intent.payment_failed
   - Copy webhook signing secret and add to Supabase secrets as STRIPE_WEBHOOK_SECRET

3. **Verify Database Tables**
   - Ensure 'payments' table exists with proper RLS policies
   - Test database connections

### Phase 2: Install EAS CLI and Configure

1. **Install EAS CLI**
   ```
   npm install -g eas-cli
   ```

2. **Login to Expo**
   ```
   eas login
   ```

3. **Configure EAS Project**
   ```
   eas init
   ```
   This will create a project ID and update your app.json

4. **Update Bundle Identifiers**
   - iOS: com.dailywhispers.app (already set in app.json)
   - Android: com.dailywhispers.app (already set in app.json)
   - Make sure these are unique and not used by other apps

### Phase 3: Build for iOS

1. **Configure iOS Build**
   ```
   eas build:configure
   ```

2. **Create iOS Production Build**
   ```
   eas build --platform ios --profile production
   ```
   
   You'll be prompted to:
   - Create or select an Apple Developer account
   - Generate or select a distribution certificate
   - Generate or select a provisioning profile

3. **Download the .ipa file** when build completes

### Phase 4: Build for Android

1. **Create Android Production Build (AAB for Play Store)**
   ```
   eas build --platform android --profile production-aab
   ```
   
   You'll be prompted to:
   - Generate a new Android keystore (or use existing)
   - IMPORTANT: Save the keystore credentials securely!

2. **Download the .aab file** when build completes

### Phase 5: Submit to Apple App Store

1. **Prepare App Store Connect**
   - Go to: https://appstoreconnect.apple.com
   - Create a new app
   - Fill in app information:
     * Name: Daily Whispers
     * Primary Language: English
     * Bundle ID: com.dailywhispers.app
     * SKU: dailywhispers001

2. **Upload Build**
   ```
   eas submit --platform ios --profile production
   ```
   
   Or manually upload via Transporter app or Xcode

3. **Complete App Store Listing**
   - App description
   - Keywords: daily quotes, inspirational, gift, notifications, positive affirmations
   - Screenshots (required sizes)
   - App icon
   - Privacy policy URL
   - Support URL
   - Age rating
   - Pricing: Free with in-app purchases

4. **Configure In-App Purchases**
   - Add each quote set as a consumable in-app purchase
   - Prices: $9.99, $18.99, $49.99
   - Provide descriptions and screenshots

5. **Submit for Review**
   - Answer App Review questions
   - Provide test account if needed
   - Submit

### Phase 6: Submit to Google Play Store

1. **Prepare Google Play Console**
   - Go to: https://play.google.com/console
   - Create a new app
   - Fill in app details:
     * App name: Daily Whispers
     * Default language: English
     * App or game: App
     * Free or paid: Free

2. **Upload Build**
   ```
   eas submit --platform android --profile production
   ```
   
   Or manually upload the .aab file to Internal Testing track first

3. **Complete Store Listing**
   - Short description (80 chars)
   - Full description (4000 chars)
   - Screenshots (required sizes)
   - Feature graphic (1024x500)
   - App icon
   - Privacy policy URL
   - Contact details

4. **Configure In-App Products**
   - Go to Monetize > Products > In-app products
   - Create products for each quote set
   - Set prices and descriptions

5. **Content Rating**
   - Complete the content rating questionnaire
   - Daily Whispers should receive an "Everyone" rating

6. **Release**
   - Start with Internal Testing
   - Then move to Closed Testing (optional)
   - Then Open Testing (optional)
   - Finally, Production release

### Phase 7: Testing Before Launch

1. **Test Payment Flow**
   - Use Stripe test cards to verify payment processing
   - Test successful payments
   - Test failed payments
   - Test refunds

2. **Test Notifications**
   - Verify daily notifications work correctly
   - Test notification permissions
   - Test notification content

3. **Test on Real Devices**
   - Test on multiple iOS devices (iPhone, iPad)
   - Test on multiple Android devices (various manufacturers)
   - Test different OS versions

4. **Test All Features**
   - Quote viewing
   - Sharing functionality
   - Saving to photo library
   - Gift sending
   - Demo mode

### Phase 8: Post-Launch

1. **Monitor Analytics**
   - Track downloads
   - Monitor crash reports
   - Review user feedback

2. **Monitor Payments**
   - Check Stripe dashboard for transactions
   - Monitor for payment failures
   - Handle refund requests

3. **Respond to Reviews**
   - Reply to user reviews on both stores
   - Address issues promptly

4. **Plan Updates**
   - Fix bugs based on user feedback
   - Add new quote sets
   - Improve features

## Important Notes

### Privacy Policy & Terms of Service
You MUST have these before submitting to app stores:
- Privacy Policy: Explain data collection, storage, and usage
- Terms of Service: Define user agreements and app usage terms
- Host these on a public URL (e.g., your website)

### App Store Review Guidelines
- Apple: https://developer.apple.com/app-store/review/guidelines/
- Google: https://play.google.com/about/developer-content-policy/

### Common Rejection Reasons
1. Missing privacy policy
2. Incomplete app information
3. Crashes or bugs
4. Misleading descriptions
5. In-app purchase issues
6. Missing required permissions explanations

### Stripe Live Mode Checklist
- [ ] testMode set to false in utils/paymentConfig.ts
- [ ] Live secret key in Supabase secrets
- [ ] Webhook configured with live endpoint
- [ ] Test with real cards (small amounts)
- [ ] Verify refund process works

### Version Updates
When releasing updates:
1. Increment version in app.json
2. Update buildNumber (iOS) and versionCode (Android)
3. Build new version with EAS
4. Submit to stores
5. Include "What's New" description
