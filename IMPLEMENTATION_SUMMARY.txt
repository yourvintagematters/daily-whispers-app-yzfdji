## 3. RENEWAL REMINDER EMAIL FEATURE

### Database Changes

#### Migration: `add_renewal_reminder_fields`

Added new columns to the `payments` table:
- `purchase_date`: Timestamp of purchase
- `subscription_end_date`: Automatically calculated as purchase_date + 365 days
- `renewal_reminder_sent`: Boolean flag (default: false)
- `renewal_reminder_sent_at`: Timestamp when reminder was sent

Created database trigger to automatically set subscription_end_date on new purchases.

Created index for efficient querying of payments needing reminders.

### New Edge Functions

#### 1. `send-renewal-reminder`
**Purpose**: Sends renewal reminder emails to customers

**Features**:
- Queries database for subscriptions ending in 7 days
- Sends personalized HTML emails to all recipients
- Includes 10% discount code (RENEW10)
- Marks reminders as sent to prevent duplicates
- Uses Resend API for email delivery

**Email Content**:
- Personalized greeting with recipient name
- Current subscription theme information
- Subscription end date
- Call-to-action button to renew
- Special 10% discount offer
- Professional HTML formatting with Daily Whispers branding

#### 2. `check-renewal-reminders-cron`
**Purpose**: Wrapper function for scheduled execution

**Features**:
- Designed to be called by cron job schedulers
- Triggers the send-renewal-reminder function
- Includes optional authentication via CRON_SECRET
- Can be called by GitHub Actions, external cron services, or pg_cron

### Email Service Integration

**Service**: Resend (https://resend.com)

**Setup Required**:
1. Sign up for Resend account
2. Get API key
3. Add API key to Supabase secrets as `RESEND_API_KEY`
4. Verify sender domain (or use test domain for development)

**Free Tier**: 100 emails/day, 3,000 emails/month

### Scheduling Options

#### Option 1: GitHub Actions (Recommended)
Create `.github/workflows/renewal-reminders.yml`:
```yaml
name: Send Renewal Reminders
on:
  schedule:
    - cron: '0 9 * * *'  # Daily at 9 AM UTC
  workflow_dispatch:

jobs:
  send-reminders:
    runs-on: ubuntu-latest
    steps:
      - name: Trigger Renewal Reminders
        run: |
          curl -X POST \
            -H "Authorization: Bearer ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}" \
            https://rfaqhkbbyvddaxyolxrx.supabase.co/functions/v1/check-renewal-reminders-cron
```

#### Option 2: External Cron Service
Use services like cron-job.org or EasyCron to call the endpoint daily.

#### Option 3: Supabase pg_cron
Use PostgreSQL's pg_cron extension for database-level scheduling.

### Renewal Email Features

- **Timing**: Sent 7 days before subscription expires
- **Personalization**: Includes recipient name and theme
- **Incentive**: 10% discount code (RENEW10)
- **Branding**: Professional HTML template with Daily Whispers colors
- **Call-to-Action**: Clear "Renew Your Subscription" button
- **Currency**: Displays prices in AUD

### Monitoring

Query to check renewal reminder status:
```sql
SELECT 
  COUNT(*) as total_reminders_sent,
  DATE(renewal_reminder_sent_at) as sent_date
FROM payments
WHERE renewal_reminder_sent = true
GROUP BY DATE(renewal_reminder_sent_at)
ORDER BY sent_date DESC;
```

Query to find upcoming reminders:
```sql
SELECT 
  id,
  option_name,
  subscription_end_date,
  recipients
FROM payments
WHERE 
  status = 'succeeded'
  AND renewal_reminder_sent = false
  AND subscription_end_date BETWEEN NOW() AND NOW() + INTERVAL '7 days'
ORDER BY subscription_end_date;
```
